# AWS Highly Available Web Application Architecture (Advanced)

## Overview
Deployed a highly available web application using multi-AZ architecture, load balancing, and health checks to ensure resilience, fault tolerance, and secure application delivery.

## Architecture Components
- Application Load Balancer for traffic distribution  
- EC2 instances deployed across multiple Availability Zones  
- Health checks for instance monitoring and failover  
- Security groups for controlled access  

## Resilience & Security Principles
- Multi-AZ deployment for high availability and fault tolerance  
- Automated health checks to detect and replace unhealthy instances  
- Controlled network access using security groups  
- Reduced single points of failure  

## Key Outcomes
- Improved application uptime and reliability  
- Reduced impact of infrastructure failures  
- Established resilient and secure web application architecture  
- Enhanced user experience through load-balanced traffic distribution  

## Technologies Used
- Application Load Balancer  
- Amazon EC2  
- Health checks  
- Security Groups
